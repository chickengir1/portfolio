# **Portfolio Project**

## **프로젝트 소개**

이 프로젝트는 개인 포트폴리오 웹사이트로, vite를 사용하여 구축되었습니다. 프로젝트 경험, 기술 스택 및 기타 정보를 깔끔한 UI와 직관적인 인터페이스로 제공하는 것이 목적입니다.

---

## **주요 기능**

### 1. **홈페이지**
- 개발자의 소개 및 주요 경력 정보 제공.
- 소셜 미디어 및 연락처 링크 포함.

### 2. **연락처**
- 이메일, GitHub 등의 링크 제공.

### 3. **기술 스택**
- 사용 가능한 언어 및 기술을 아이콘으로 정리.
- 각 기술의 숙련도 표시.

### 4. **프로젝트 섹션**
- 참여한 프로젝트 목록을 카테고리별로 정리.
- 프로젝트별 상세 설명, 기술 스택, GitHub 링크, 배포 URL 제공.

---

## **기술 스택**

### **프론트엔드**
- **Vite**: 컴포넌트 기반 UI 구축.
- **Tailwind Css**: 스타일링 및 UI 관리.
- **TypeScript**: 코드 가독성과 유지보수성 향상.

---

## **아키텍처**

### 디렉토리 구조

```
- components/
  - anime/
  - chip/
  - layout/
- hooks/
- utils/
- App.tsx
- index.css
- main.tsx
- vite-env.d.ts
```

#### 디렉토리 상세 설명

- **components/**: 재사용 가능한 UI 컴포넌트들이 위치한 디렉토리입니다.
  - **anime/**: 애니메이션과 관련된 컴포넌트입니다. 인터랙티브한 요소들을 구현하였습니다.
  - **chip/**: 기술 스택이나 프로젝트 태그 등을 표시하기 위한 '칩' 형태의 컴포넌트들을 포함하고 있습니다.
  - **layout/**: 페이지의 기본 레이아웃과 공통 구조를 담당하는 컴포넌트들이 위치합니다.

- **hooks/**: 커스텀 훅을 저장하는 디렉토리입니다. 상태 관리나 공통 로직을 재사용 가능한 형태로 분리하였습니다.

- **utils/**: 유틸리티 함수들이 모여 있는 디렉토리로, 데이터 포맷팅이나 공통 상수 등을 관리합니다.
- 
### 컴포넌트 아키텍처

이번 프로젝트에서는 **애니메이션**과 **칩(Chip)** 컴포넌트를 중심으로 컴포넌트를 구성하였습니다.

- **애니메이션 컴포넌트 (`components/anime/`)**:
  - 사용자 경험을 향상시키기 위해 페이지 전환이나 요소 등장 시 애니메이션 효과를 추가하였습니다.
  - React의 라이프사이클과 CSS 애니메이션을 활용하여 부드러운 인터랙션을 구현하였습니다.

- **칩 컴포넌트 (`components/chip/`)**:
  - 기술 스택, 프로젝트 카테고리 등을 시각적으로 구분하기 위해 칩 형태의 컴포넌트를 사용하였습니다.

- **레이아웃 컴포넌트 (`components/layout/`)**:
  - 전체 페이지의 일관된 구조를 유지하기 위해 레이아웃 컴포넌트를 활용하였습니다.

### 관심사 분리와 모듈화

- **재사용 가능한 컴포넌트 설계**:
  - 컴포넌트들을 기능별로 분리하여 필요한 곳에서 쉽게 가져다 사용할 수 있도록 하였습니다.
  - 버튼, 입력 폼 등 기본적인 UI 요소를 별도의 컴포넌트로 만들어 일관된 디자인을 유지하였습니다.

- **커스텀 훅 활용 (`hooks/`)**:
  - 반복되는 로직이나 상태 관리는 커스텀 훅으로 분리하여 코드의 재사용성을 높였습니다.
  - 예를 들어, 화면 크기에 따라 반응형으로 동작하는 훅이나 스크롤 위치를 감지하는 훅 등을 구현하였습니다.

- **유틸리티 함수 (`utils/`)**:
  - 프로젝트 전반에서 사용되는 공통 함수나 상수를 `utils/` 디렉토리에 모아 관리하였습니다.
  - 날짜 형식 변환이나 문자열 처리와 같은 기능을 유틸리티로 제공하여 코드 중복을 방지하였습니다.

### 커밋 컨벤션

- **커밋 컨벤션 준수**:
  - 일관된 커밋 메시지 컨벤션을 사용하여 변경 사항을 명확하게 기록하였습니다.
  - 예시:
    - `feat: 프로젝트 페이지 구현 완료`
    - `fix: 미사용 파일 삭제`
    - `feat: 폴더 클릭 시 GitHub로 이동하는 로직 추가`


### 주요 구현 사항

- **반응형 디자인 구현**:
  - **Tailwind CSS**를 활용하여 모바일부터 데스크톱까지 다양한 화면 크기에 대응하는 반응형 웹사이트를 제작하였습니다.
  - 유틸리티 클래스의 사용으로 일관된 스타일링과 빠른 개발이 가능하였습니다.

- **3D 그래픽 요소 적용**:
  - **Three.js**를 사용하여 포트폴리오 사이트에 3D 애니메이션을 추가하여 시각적 효과를 강화하였습니다.
  - 사용자에게 인상적인 첫인상을 줄 수 있도록 메인 페이지에 적용하였습니다.

- **사용자 경험 개선**:
  - 애니메이션과 인터랙티브 요소를 통해 사용자에게 즐거운 탐색 경험을 제공하였습니다.

- **배포 및 호스팅**:
  - **GitHub Pages**를 통해 애플리케이션을 배포하여 안정적인 호스팅 환경을 구축하였습니다.
  - 지속적인 업데이트와 유지보수가 용이하도록 설정하였습니다.
